ARRAY LISTS ▼

What it is:

An ArrayList is a resizable array implementation in Java that allows dynamic storage and manipulation of elements, enabling efficient addition and removal after the compilation phase, while storing reference types.

When to Use an Array List

  • Where arrays are a fixed allocation size, ArrayLists can grow or shrink dynamically to contain desired elements
  • Does not support primitive data types; only use reference types
  • Supports random access; elements can be accessed directly rather than in a fixed sequence
  • Runtime Complexity:

Simple ArrayList Methods:

  • add(element): adds element at the end of the list
  • remove(index): removes element at desired index
  • get(index): returns element at desired index
  • set(index, element): sets desired index with desired element
  • size(): returns the size of the arraylist

Implementation:

        
            
        
import java.util.ArrayList;    
        
 public class Main {
            
    public static void main(String[] args) {
                    
    ArrayList grocery = new ArrayList(); 
                                                        
        grocery.add("milk");
         grocery.add("eggs");
         grocery.add("cheese");
                    
         grocery.set(0, "low fat milk");
         grocery.remove(2);
         grocery.clear();
                    
        for(int i = 0; i < grocery.size(); i++) {     
            System.out.println(grocery.get(i));
        }
    }
}
LINKED LISTS ▼

A LinkedList is a linear data structure where each element (node) contains a reference to the next node, allowing efficient insertion and deletion operations at any position.

STACKS ▼

A Stack is a Last-In-First-Out (LIFO) data structure used for operations like recursion, undo functionality, and expression evaluation.

QUEUES ▼

A Queue is a First-In-First-Out (FIFO) data structure used in scheduling, buffering, and asynchronous tasks.

PRIORITY QUEUES ▼

A Priority Queue assigns priorities to elements, ensuring higher-priority elements are dequeued before lower-priority ones, regardless of insertion order.

BINARY TREES ▼

A Binary Tree is a hierarchical data structure with nodes having up to two children, used in searching, sorting, and hierarchical data representation.

HEAPS ▼

A Heap is a complete binary tree used for implementing priority queues, with the heap property ensuring parent nodes are ordered with respect to their children.

HASH TABLES ▼

A Hash Table maps keys to values for efficient lookups, insertions, and deletions using a hash function to index data.

GRAPHS ▼

A Graph is a collection of nodes (vertices) and edges, used to model pairwise relations in applications like networking, search engines, and transportation.
