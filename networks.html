1.2 NETWORK APPLIANCES, APPLICATIONS, AND FUNCTIONS ▼
  • Physical and Virtual Appliances
    • Router: A router is a device that forwards data packets between different networks, directing traffic to its destination. It is also known as a layer 3 device
    • Switch: A switch is a device that connects devices within a single network and directs data to the correct destination device based on MAC addresses.
    • Firewall: A firewall is a security device that monitors and controls incoming and outgoing network traffic based on predetermined security rules to protect networks from unauthorized access.
    • Intrusion Detection System (IDS) / Intrusion Prevention System (IPS): An Intrusion Detection System (IDS) monitors network traffic for suspicious activity or potential threats, while an Intrusion Prevention System (IPS) actively blocks or prevents detected threats from harming the network.
    • Load Balancer: a device or software that distributes incoming network traffic across multiple servers to ensure optimal performance, reliability, and scalability of applications.
    • Proxy: a server that acts as an intermediary between a user's device and the internet, forwarding requests and responses to improve security, privacy, or performance.
    • Network-attached Storage (NAS): a dedicated storage device connected to a network, allowing multiple users and devices to access and share files over the network.
    • Storage Area Network (SAN:) a high-speed network that provides block-level access to data storage, allowing multiple servers to access shared storage devices.
    • Wireless
      • Access Point (AP): a device that allows wireless devices to connect to a wired network using Wi-Fi, extending the network's coverage.
      • Controller: a device or software that manages and coordinates multiple wireless access points (APs) in a network, ensuring consistent configuration, security, and performance across all APs.
  • Applications
    • Content Delivery Network (CDN): a system of distributed servers that deliver web content and media to users based on their geographic location, improving speed and reliability.
  • Functions
    • Virtual Private Network (VPN): a service that creates a secure, encrypted connection over the internet, allowing users to access a private network remotely and protect their online privacy
    • Quality of Service (QoS): a set of technologies that manage and prioritize network traffic
    • Time to Live (TTL): a value in a network packet that specifies the maximum number of hops (or routers) the packet can pass through before being discarded, helping prevent endless loops in routing.
1.3 CLOUD CONCEPTS ▼
  • Network Functions Virtualization (NFV): the use of virtualization technology to deliver network services, such as firewalls or load balancers, via software instead of traditional hardware appliances.
  • Virtual Private Cloud (VPC): a private, isolated network within a public cloud that allows users to securely run resources and applications with control over network configurations.
  • Network Security Groups: a set of rules that control inbound and outbound traffic to network resources, helping secure virtual machines and other services in a cloud environment
  • Network Security Lists: a set of rules used to control inbound and outbound traffic at the subnet level in a cloud network, providing an additional layer of security for cloud resources.
  • Cloud Gateways
    • Internet Gateway: a device that connects a Virtual Private Cloud (VPC) to the internet, allowing communication between resources in the VPC and external networks
    • Network Address Translation (NAT) Gateway: a service that allows resources in a private network to access the internet while keeping their private IP addresses hidden, by translating them to a public IP address.
  • Cloud Connectivity Options
    • VPN: a method that allows secure, encrypted communication between an on-premises network and a cloud network
    • Direct Connect: a service that provides a private, direct link between your company's network and the cloud, making the connection faster and more stable than using the public internet.
  • Deployment Models
    • Public: Public being accessible to everyone over the internet
    • Private Private offering dedicated resources for a specific organization
    • Hybrid Hybrid combining both public and private cloud resources for greater flexibility
  • Service Models
    • Software as a Service (SaaS) Public services
    • Infrastructure as a Service (IaaS): Low level, private services
    • Platform as a Service (PaaS): Hybrid, room for developers
  • Scalability: the ability of a system to handle increased workload or traffic by adding resources, such as more servers or storage
  • Elasticity: ability of a system to automatically scale resources up or down based on current demand, ensuring efficient use of resources and cost-effectiveness.
  • Multitenancy: a single software or service is used by multiple users or organizations, but each one has its own separate space within the system
1.4 PORTS, PROTICALS, AND TRAFFIC TYPES▼
  • Internet Protocol (IP) types
    • Internet Control Message Protocol (ICMP): a network protocol used for error reporting and diagnostics, such as sending "ping" requests to check the availability of a network device.
    • Transmission Control Protocol (TCP)
    • User Datagram Protocol (UDP)
    • Generic Routing Encapsulation (GRE): a tunneling protocol used to encapsulate and route different types of network traffic (like IPv4, IPv6) over a point-to-point connection, often used for creating virtual private networks (VPNs).
    • Internet Protocol Security (IPSec): a suite of protocols used to secure internet communications by encrypting and authenticating data at the IP layer, ensuring privacy and integrity for network traffic
      • Authentication Header (AH): a component of IPSec that provides data integrity and authentication for IP packets, ensuring that the data has not been tampered with during transmission, but it does not provide encryption.
      • Encapsulating Security Payload (ESP): a component of IPSec that provides encryption for data, ensuring both confidentiality and integrity of IP packets while also supporting authentication.
      • Internet Key Exchange (IKE): a protocol used to securely exchange encryption keys between devices to establish a secure VPN connection.
  • Traffic types
    • Unicast
    • Multicast
    • Anycast: nearest or best destination from a group of potential receivers
    • Broadcast: all

1.6 NETWORK TOPOLOGIES AND ARCHITECTURES ▼
  • Mesh: a type of network setup where each device or node is connected to multiple other devices, allowing for direct, dynamic communication between them and creating multiple paths for data to travel
  • Hybrid: more than one network topology
  • Star/Hub and Spoke: a network topology where all devices are connected to a central hub, which acts as the main point of communication, with each device (spoke) only directly communicating with the hub.
  • Spine and Leaf: a network architecture where the spine switches connect to all leaf switches, creating a high-bandwidth, low-latency network where each leaf switch is connected to every spine switch for optimal data flow.
  • Point to Point: a network topology where two devices are directly connected to each other, enabling a dedicated communication link between them.
  • Three-Tier Hierarchical Model
    • Core: responsible for high-speed, reliable data transmission across the network and connects different distribution layers or data centers.
    • Distribution: acts as an intermediary between the Core and Access layers, handling routing, filtering, and traffic management between different network segments.
    • Access: connects end-user devices or computers to the network, providing the interface for users to access network resources.
  • Collapsed Core: a network architecture where the Core and Distribution layers are combined into a single layer, simplifying the network design and reducing hardware costs while still providing high performance and scalability.
  • Traffic Flows
    • North-South: User and cloud
    • East-West:User and datacenter / network
1.7 IPv4 NETWORK ADDRESSING▼
  • Public vs. Private
    • Automatic Private IP Addressing (APIPA): a feature that allows a device to assign itself an IP address automatically when a DHCP server is unavailable.
    • RFC1918: a set of private IP address ranges that are reserved for use within private networks and are not routable on the public internet.
    • Loopback/localhost: The most commonly used address is 127.0.0.1.
  • Subnetting
    • Variable Length Subnet Mask (VLSM): a technique that allows different subnet masks to be applied to different subnets within the same network, enabling more efficient use of IP address space.
    • Classless Inter-domain Routing (CIDR): a method for allocating and routing IP addresses that allows more flexible subnetting by using variable-length subnet masks instead of fixed classes
  • IPv4 Address Classes
    • Class A
    • Class B
    • Class C
    • Class D
    • Class E

3.0 NETWORK OPERATIONS▼
3.1 ORGANIZATIONAL PROCESSES AND PROCEDURES ▼
  • Documentation
    • Physical vs. logical diagrams: Diagrams showing the physical and logical layout of systems.
    • Rack diagrams: Diagrams showing the arrangement of equipment in a server rack.
    • Cable maps and diagrams: Diagrams detailing the cabling used in a network.
    • Network diagrams
      • Layer 1: Physical layout of the network (cabling, devices).
      • Layer 2: Data link layer, showing switches and data paths.
      • Layer 3: Network layer, showing routing and IP addresses.
    • Asset inventory
      • Hardware: List of physical equipment like computers, routers, etc.
      • Software: List of software applications used within the organization.
      • Licensing: Tracking of software licenses for compliance.
      • Warranty support: Information about warranties for devices or systems.
    • IP address management (IPAM): Managing and assigning IP addresses in a network.
    • Service-level agreement (SLA): Contract that defines the level of service expected from a provider.
    • Wireless survey/heat map: A map showing the coverage and performance of a wireless network.
  • Life-cycle management
    • End-of-life (EOL): Point at which a product is no longer supported by the manufacturer.
    • End-of-support (EOS): When a product no longer receives official support or updates.
    • Software management
      • Patches and bug fixes: Updates made to fix security holes or issues in software.
      • Operating system (OS): Management of the operating system to ensure it is up-to-date.
      • Firmware: Software embedded into hardware that may need updates for functionality.
    • Decommissioning: The process of taking a system or equipment out of service.
  • Change management
    • Request process tracking/service request: Tracking requests for changes or support to ensure proper follow-through.
  • Configuration management
    • Production configuration: The current, working setup of hardware and software in a live environment.
    • Backup configuration: A saved copy of the production configuration for recovery purposes.
    • Baseline/golden configuration: The standard, optimal configuration to which systems should adhere.
3.2 NETWORK MONITORING TECHNOLOGIES ▼
  • Methods
    • SNMP: Simple Network Management Protocol used for network device management.
      • Traps: Notifications sent by devices to inform management systems of specific events.
      • Management information base (MIB): A database used by SNMP to store information about the network devices.
      • Versions: Different versions of SNMP with varying features and security levels.
        • v2c: The second version of SNMP, offering improved features and performance over v1.
        • v3: The third version of SNMP, focusing on security and encryption.
      • Community strings: Authentication strings used in SNMP to access network devices.
      • Authentication: Security method used in SNMPv3 to ensure that data is being sent to and from authorized devices.
    • Flow data: Information about network traffic flow, often used for traffic analysis and monitoring.
    • Packet capture: The process of intercepting and analyzing data packets being transmitted across the network.
    • Baseline metrics: Standardized performance data used to compare and detect anomalies in the network.
      • Anomaly alerting/notification: Alerts generated when network traffic deviates from established baseline metrics.
    • Log aggregation: Collecting logs from multiple sources for centralized analysis and management.
      • Syslog collector: A tool that gathers log messages from devices and systems in a network.
      • Security information and event management (SIEM): A system used to monitor and analyze security events and logs across a network.
    • Application programming interface (API) integration: The ability to connect network monitoring tools with other systems or services via APIs for enhanced functionality.
    • Port mirroring: A method of monitoring network traffic by duplicating data packets from one port to another for analysis.
  • Solutions
    • Network discovery: The process of identifying devices and systems on a network.
      • Ad hoc: A one-time network discovery process, typically initiated manually.
      • Scheduled: A recurring network discovery process, typically automated to run at set intervals.
    • Traffic analysis: The process of examining network traffic patterns to assess performance or identify issues.
    • Performance monitoring: Tracking network performance to ensure it meets defined standards or service level agreements (SLAs).
    • Availability monitoring: Ensuring that network devices and services are accessible and operating as expected.
    • Configuration monitoring: Monitoring and managing network configurations to ensure they are correct and secure.
3.3 DISASTER RECOVERY ▼
  • DR Metrics
    • Recovery Point Objective (RPO): The maximum acceptable amount of data loss measured in time.
    • Recovery Time Objective (RTO): The maximum acceptable amount of time for the system to be restored after a disaster.
    • Mean Time to Repair (MTTR): The average time required to repair and restore a system after a failure.
    • Mean Time Between Failures (MTBF): The average time between consecutive failures of a system or component.
  • DR Sites
    • Cold Site: A location with infrastructure but no immediate hardware or data backup available; requires setup during an outage.
    • Warm Site: A location with infrastructure and some pre-configured hardware, but data backups still need to be restored during recovery.
    • Hot Site: A fully equipped and operational backup site with current data and systems available for immediate failover.
  • High-Availability Approaches
    • Active-Active: Multiple systems or locations running simultaneously to provide load balancing and fault tolerance.
    • Active-Passive: One system actively provides services while the other remains as a standby in case of failure.
  • Testing
    • Tabletop Exercises: A simulated disaster scenario where participants discuss and review procedures in a low-stress, non-technical environment.
    • Validation Tests: Practical tests designed to validate the functionality of the DR plan, often involving failover and recovery procedures.
3.4 IPv4 AND IPv6▼
  • Dynamic Addressing - DHCP
    • Reservations: Assigning specific IP addresses to specific devices based on MAC addresses.
    • Scope: The range of IP addresses available for DHCP assignment.
    • Lease Time: The duration for which an IP address is assigned to a device before it needs to be renewed.
    • Options: Additional configuration parameters provided by the DHCP server, such as DNS server or gateway information.
    • Relay/IP Helper: A mechanism to forward DHCP requests between clients and servers on different subnets.
    • Exclusions: IP addresses within the scope that are excluded from being assigned to clients by DHCP.
  • Stateless Address Autoconfiguration (SLAAC): A method used by IPv6 devices to configure their own IP addresses without needing a DHCP server.
  • Name Resolution - DNS
    • Domain Name Security Extensions (DNSSEC): A suite of extensions to DNS to add security, such as ensuring data integrity and authenticity.
    • DNS over HTTPS (DoH) and DNS over TLS (DoT): Protocols that encrypt DNS queries to increase privacy and security.
    • Record Types
      • Address (A): Maps a domain to an IPv4 address.
      • AAAA: Maps a domain to an IPv6 address.
      • Canonical Name (CNAME): Aliases one domain to another domain.
      • Mail Exchange (MX): Defines mail servers for a domain.
      • Text (TXT): Stores arbitrary text, often used for verification and policies.
      • Nameserver (NS): Specifies authoritative name servers for a domain.
      • Pointer (PTR): Used for reverse DNS lookups, mapping IP addresses to domain names.
    • Zone Types
      • Forward: A DNS zone where requests are answered with IP addresses or names.
      • Reverse: A DNS zone used for reverse lookups, mapping IP addresses to domain names.
      • Authoritative vs. Non-Authoritative: Authoritative servers provide the definitive answers for their domain, while non-authoritative servers relay answers from other sources.
      • Primary vs. Secondary: Primary DNS servers hold the original records, while secondary servers hold copies for redundancy.
      • Recursive: A type of DNS query where the server performs full resolution on behalf of the client.
  • Hosts File: A local file used to map hostnames to IP addresses, typically for systems to resolve names without needing DNS queries.
  • Time Protocols
    • Network Time Protocol (NTP): A protocol used to synchronize system clocks over a network.
    • Precision Time Protocol (PTP): A more precise time synchronization protocol, typically used in industrial and high-performance environments.
    • Network Time Security (NTS): An extension to NTP that adds security features, ensuring the integrity of time synchronization against attacks.
3.5 NETWORK ACCEESS AND MANAGEMENT METHODS ▼
  • Site-to-Site VPN: A VPN connection between two or more networks, allowing them to securely communicate over the internet.
  • Client-to-Site VPN
    • Clientless: A VPN connection that does not require the installation of client software, often using a web browser to establish the connection.
    • Split Tunnel vs. Full Tunnel
      • Split Tunnel: Only traffic destined for the corporate network is routed through the VPN, while other internet traffic bypasses the VPN.
      • Full Tunnel: All traffic, including internet traffic, is routed through the VPN for enhanced security and privacy.
  • Connection Methods
    • SSH: A secure, encrypted protocol for remote access to network devices and systems.
    • Graphical User Interface (GUI): A user-friendly interface with graphical elements for managing and interacting with systems and devices.
    • API: An interface that allows different software systems to communicate and interact programmatically.
    • Console: A text-based interface for directly managing network devices or systems, often used for initial configuration and troubleshooting.
  • Jump Box/Host: A secure, intermediary server used to access and manage devices in a network, often placed in a DMZ (demilitarized zone) for added security.
  • In-band vs. Out-of-Band Management
    • In-band Management: Management of network devices using the same network as the device's normal traffic, typically via SSH or a web interface.
    • Out-of-Band Management: Management of network devices using a dedicated management network or port, separate from the production traffic, offering enhanced security and availability during network failures.
